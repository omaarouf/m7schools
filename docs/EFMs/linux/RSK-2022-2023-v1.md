## Partie Théorique (13,5 pts) — 1,5 pt par question

### Question 1 — Rôle du fichier `/etc/hosts` (1,5 pt)

<details>
<summary>Voir la réponse</summary>

Le fichier `/etc/hosts` est un fichier de résolution de noms **local** — il associe des noms d'hôtes à des adresses IP sans passer par un serveur DNS. Il est consulté en **priorité** avant les requêtes DNS.

```bash
# Exemple de contenu
127.0.0.1       localhost
192.168.2.1     ServDns.societe.com  ServDns
192.168.2.2     poste-client
```

Utilisation typique : accès rapide à des machines du réseau local, redirection de noms, environnements sans DNS.

</details>

### Question 3 — Utilité de la commande `arp` (1,5 pt)

<details>
<summary>Voir la réponse</summary>

La commande `arp` (Address Resolution Protocol) permet de :
- Afficher la **table ARP** du système (correspondances IP ↔ MAC connues)
- Ajouter ou supprimer des entrées ARP manuellement

```bash
arp -a              # Afficher toutes les entrées ARP
arp -n              # Affichage numérique (sans résolution DNS)
arp -d 192.168.2.5  # Supprimer une entrée
```

</details>

---

### Question 4 — À quoi sert le routage ? (1,5 pt)

<details>
<summary>Voir la réponse</summary>

Le routage permet d'**acheminer les paquets IP** d'un réseau source vers un réseau de destination, potentiellement à travers plusieurs réseaux intermédiaires. Le routeur consulte sa **table de routage** pour déterminer par quelle interface et vers quel prochain saut (next-hop) envoyer chaque paquet.

Sans routage, la communication entre réseaux différents est impossible.

</details>

---

### Question 5 — Différence entre routage statique et dynamique (1,5 pt)

<details>
<summary>Voir la réponse</summary>

| Critère | Routage statique | Routage dynamique |
|---|---|---|
| Configuration | Manuelle par l'admin | Automatique via protocole (RIP, OSPF, EIGRP) |
| Adaptation aux pannes | Non — intervention manuelle requise | Oui — recalcul automatique |
| Charge CPU/bande passante | Nulle | Faible (échanges entre routeurs) |
| Complexité | Simple | Plus complexe |
| Usage | Petits réseaux | Grands réseaux |

</details>

### Question 6 — Ajouter une route statique vers `192.168.0.0/24` via `100.15.120.5` (1,5 pt)

<details>
<summary>Voir la réponse</summary>

```bash
# Temporaire (perdu au redémarrage)
sudo ip route add 192.168.0.0/24 via 100.15.120.5

# Permanent avec nmcli
sudo nmcli con mod eth0 +ipv4.routes "192.168.0.0/24 100.15.120.5"
sudo nmcli con up eth0

# Vérifier
ip route show
```

</details>

### Question 8 — Utilité de la commande `netstat` (1,5 pt)

<details>
<summary>Voir la réponse</summary>

`netstat` affiche les **connexions réseau actives**, les ports en écoute, les tables de routage et les statistiques des interfaces.

```bash
netstat -tuln      # Ports TCP/UDP en écoute
netstat -an        # Toutes les connexions
netstat -r         # Table de routage
netstat -s         # Statistiques par protocole
```

:::info
Sur les systèmes récents, `netstat` est remplacé par `ss` (plus rapide) :
```bash
ss -tuln           # Equivalent moderne de netstat -tuln
```
:::

</details>

### Question 9 — Avantages et inconvénients de RAID0 et RAID1 (1,5 pt)

<details>
<summary>Voir la réponse</summary>

**RAID 0 (Striping) :**

| Avantages | Inconvénients |
|---|---|
| Meilleures performances (lecture/écriture) | Aucune redondance |
| Utilisation de 100% de la capacité | Perte totale si 1 disque tombe |
| Simple à mettre en place | Pas de tolérance aux pannes |

**RAID 1 (Mirroring) :**

| Avantages | Inconvénients |
|---|---|
| Redondance complète (tolérance 1 panne) | Capacité utile = 50% |
| Lecture plus rapide (depuis les 2 disques) | Plus coûteux (2x les disques) |
| Reconstruction simple | Écriture légèrement plus lente |

</details>

**1a. Configurer l'interface réseau `eth0` avec la première adresse du sous-réseau `192.168.2.0/25`. (2 pts)**

<details>
<summary>Voir la réponse</summary>

La première adresse utilisable de `192.168.2.0/25` est `192.168.2.1` (masque `255.255.255.128`).

```bash
ip a show eth0
```

**4. Créer la zone DNS principale directe `societe.com` et sa zone inversée avec notification, transfert et mise à jour dynamique. (4 pts)**

Le serveur secondaire est `192.168.2.2`, fichier zone directe : `société.dir`.

<details>
<summary>Voir la réponse</summary>

Dans `/etc/named.conf` :

```bash
# Zone directe
zone "societe.com" IN {
    type master;
    file "societe.dir";
    allow-update { 192.168.2.2; };
    allow-transfer { 192.168.2.2; };
    notify yes;
};

# Zone inversée
zone "2.168.192.in-addr.arpa" IN {
    type master;
    file "societe.inv";
    allow-update { 192.168.2.2; };
    allow-transfer { 192.168.2.2; };
    notify yes;
};
```

</details>

**5. Créer le fichier de zone directe `société.dir` avec les enregistrements A, NS et SOA pour ServDns. (3 pts)**

```bash
            86400 )     ; Minimum TTL

; Enregistrement A
ServDns IN  A       192.168.2.1
```

</details>

### II — Configuration du serveur DHCP (4,5 pts)

---

**1. Créer une étendue DHCP de `192.168.2.20` à `192.168.2.250` en excluant `192.168.2.45` à `192.168.2.70`. (4 pts)**

Options : Passerelle `192.168.2.254` · DNS `192.168.2.1` · Domaine `ofppt.ma` · Bail minimal 1 heure

<details>
<summary>Voir la réponse</summary>

```bash
# /etc/dhcp/dhcpd.conf

# Adresses exclues
subnet 192.168.2.0 netmask 255.255.255.128 {

    # Plage 1 : avant la zone exclue
    range 192.168.2.20 192.168.2.44;

    # Plage 2 : après la zone exclue
    range 192.168.2.71 192.168.2.250;

    option routers 192.168.2.254;
    option domain-name-servers 192.168.2.1;
    option domain-name "ofppt.ma";
    default-lease-time 3600;
    max-lease-time 7200;
}
```

Démarrer le service :

```bash
# Fedora
sudo systemctl restart dhcpd
sudo systemctl enable dhcpd

# Ubuntu
sudo systemctl restart isc-dhcp-server
sudo systemctl enable isc-dhcp-server
```

</details>

**2. Regrouper les trois disques en un seul disque logique RAID1 avec un disque de secours. (3 pts)**

<details>
<summary>Voir la réponse</summary>

Avec 3 disques (`/dev/sdb`, `/dev/sdc`, `/dev/sdd`) :
- RAID 1 nécessite 2 disques actifs
- Le 3ème devient le **spare** (disque de secours)

```bash
sudo mdadm --create /dev/md0 \
  --level=1 \
  --raid-devices=2 \
  /dev/sdb /dev/sdc \
  --spare-devices=1 /dev/sdd

# Surveiller la synchronisation
watch cat /proc/mdstat

# Sauvegarder la configuration
sudo mdadm --detail --scan | sudo tee -a /etc/mdadm/mdadm.conf
sudo update-initramfs -u   # Ubuntu
# sudo dracut --force      # Fedora
```

</details>

<details>
<summary>Voir la réponse</summary>

```bash
sudo mkdir -p /srv/nfs/formation
```

</details>

---

**2. Donner les droits de lecture, écriture et exécution à tout le monde. (0,5 pt)**

<details>
<summary>Voir la réponse</summary>

```bash
sudo chmod 777 /srv/nfs/formation

# Vérifier
ls -ld /srv/nfs/formation
```

</details>

---

**3. Partager le dossier avec l'ordinateur d'adresse `192.168.2.100` en lecture seule. (2,5 pts)**

<details>
<summary>Voir la réponse</summary>

Dans `/etc/exports` :

```bash
/srv/nfs/formation    192.168.2.100(ro,sync,no_root_squash)
```

Appliquer et démarrer NFS :

```bash
# Appliquer les exports
sudo exportfs -ra

# Fedora / RHEL
sudo systemctl start nfs-server
sudo systemctl enable nfs-server

# Ubuntu
sudo systemctl start nfs-kernel-server
sudo systemctl enable nfs-kernel-server

# Vérifier
sudo exportfs -v

# Autoriser dans le pare-feu (Fedora)
sudo firewall-cmd --add-service=nfs --permanent
sudo firewall-cmd --reload
```

</details>

---

**4. Accéder au dossier partagé depuis l'ordinateur d'adresse `192.168.2.100`. (2,5 pts)**

<details>
<summary>Voir la réponse</summary>

Sur le **client** (`192.168.2.100`) :

```bash
# Installer le client NFS
sudo apt install nfs-common -y       # Ubuntu
sudo dnf install nfs-utils -y        # Fedora

# Créer le point de montage
sudo mkdir -p /mnt/formation

# Monter le partage NFS
sudo mount 192.168.2.1:/srv/nfs/formation /mnt/formation

# Vérifier
df -h /mnt/formation
ls /mnt/formation

# Montage permanent dans /etc/fstab
echo '192.168.2.1:/srv/nfs/formation /mnt/formation nfs defaults 0 0' | sudo tee -a /etc/fstab
```

</details>
