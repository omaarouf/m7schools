## Partie Théorique — Exercice 1 (8 pts)

### Question 1 — `/dev/hdd2` représente : (2 pts)

- a. Le deuxième périphérique du disque dur N° 4 de votre système Linux
- b. La deuxième partition de votre deuxième disque maître de votre système Linux
- c. La deuxième partition de votre deuxième disque esclave de votre système Linux
- d. Aucune de ces réponses

<details>
<summary>Voir la réponse</summary>

**Réponse : c** — La deuxième partition de votre deuxième disque esclave de votre système Linux.

Explication de la nomenclature `/dev/hdd2` :

Dans les anciens systèmes (IDE / PATA) :

Disque maître (master) : disque principal sur la nappe

Disque esclave (slave) : disque secondaire sur la même nappe

Linux nomme ces disques ainsi :

1er disque maître → /dev/hda

1er disque esclave → /dev/hdb

2e disque maître → /dev/hdc

2e disque esclave → /dev/hdd

* **Deuxième disque maître** : c’est le 2ᵉ disque principal du système → `/dev/hdc`
  → sa 2ᵉ partition : **`/dev/hdc2`**

* **Deuxième disque esclave** : c’est le 2ᵉ disque secondaire → `/dev/hdd`
  → sa 2ᵉ partition : **`/dev/hdd2`**

**Différence simple :**

* `hdc` = disque principal (maître)
* `hdd` = disque secondaire (esclave)
* `2` = numéro de la partition


</details>

### Question 3 — Expliquer le fonctionnement d'un RAID de niveau 1 (2 pts)

<details>
<summary>Voir la réponse</summary>

Le **RAID 1** (mirroring / mise en miroir) fonctionne ainsi :

- Les données sont **écrites simultanément** sur deux disques ou plus de façon identique
- Chaque disque contient une **copie exacte** des données
- En cas de panne d'un disque, les données sont toujours disponibles sur le ou les disques restants
- La **capacité utile** est égale à la taille d'un seul disque (50% de la capacité totale)
- Le RAID 1 améliore la **tolérance aux pannes** mais pas les performances en écriture

**Exemple :** 2 disques de 500 Go → capacité utile = 500 Go, tolérance = 1 panne disque.

</details>

### Question 4 — À quoi sert le fichier `/etc/fstab` ? (2 pts)

<details>
<summary>Voir la réponse</summary>

Le fichier `/etc/fstab` (File System TABle) est lu au **démarrage du système** et contient la liste des systèmes de fichiers à monter automatiquement.

Chaque ligne définit :
- Le **périphérique ou UUID** à monter
- Le **point de montage** (répertoire cible)
- Le **type de système de fichiers** (ext4, xfs, nfs...)
- Les **options de montage** (defaults, ro, noauto...)
- Les paramètres **dump** et **fsck** (sauvegarde et vérification au boot)

```bash
# Exemple de ligne fstab
/dev/vg_data/lv_web  /var/www  ext4  defaults  0  2
```

Sans cette entrée, le montage serait perdu à chaque redémarrage.

</details>

## Partie Pratique — Exercice 2 : Serveur WEB Apache (14 pts)

:::info Contexte
Vous êtes administrateur d'un réseau avec un **Serveur WEB** (`10.10.0.1/29`) et son **Client WEB** (`10.10.0.2/29`). Votre tâche est de configurer le serveur Apache pour servir une page `index.html`.
:::

---

### Question 1 — Configurer les adresses IP des deux machines en ligne de commande (2 pts)

<details>
<summary>Voir la réponse</summary>

**Sur le Serveur WEB (`10.10.0.1/29`) :**

```bash
sudo nmcli con mod ens33 ipv4.method manual \
  ipv4.addresses 10.10.0.1/29

sudo nmcli con up ens33
ip a show ens33
```

**Sur le Client WEB (`10.10.0.2/29`) :**

```bash
sudo nmcli con mod ens33 ipv4.method manual \
  ipv4.addresses 10.10.0.2/29

sudo nmcli con up ens33
ip a show ens33
```

:::tip
Le masque `/29` correspond à `255.255.255.248` — la plage réseau `10.10.0.0/29` contient 8 adresses dont 6 utilisables (10.10.0.1 à 10.10.0.6).
:::

</details>

### Question 4 — Ligne à modifier dans `httpd.conf` pour l'écoute sur le port 80 (3 pts)

<details>
<summary>Voir la réponse</summary>

Dans le fichier `/etc/httpd/conf/httpd.conf` (Fedora) ou `/etc/apache2/ports.conf` (Ubuntu), la directive à modifier est :

```apache
Listen 80
```

Pour spécifier une adresse d'écoute précise :

```apache
Listen 10.10.0.1:80
```

:::info
Par défaut, `Listen 80` fait écouter Apache sur **toutes les interfaces** du serveur sur le port 80. En spécifiant l'IP, on limite l'écoute à une interface précise.
:::

</details>

### Question 2 — Configuration de la zone directe `efm.local` dans `named.conf` (6 pts)

Paramètres :
- Nom de zone : `efm.local`
- Type : principal (master)
- Fichier de zone : `efm_local.db`
- Autoriser la mise à jour dynamique avec le serveur DHCP `10.10.0.30`
- Autoriser le transfert vers `10.10.0.30`
- Activer la notification

<details>
<summary>Voir la réponse</summary>

Dans `/etc/named.conf` (Fedora) ou `/etc/bind/named.conf.local` (Ubuntu) :

```bash
zone "efm.local" IN {
    type master;
    file "efm_local.db";
    allow-update { 10.10.0.30; };
    allow-transfer { 10.10.0.30; };
    notify yes;
};
```

Explication de chaque directive :

| Directive | Rôle |
|---|---|
| `type master` | Ce serveur est le serveur DNS primaire (principal) |
| `file "efm_local.db"` | Fichier de zone stocké dans `/var/named/` |
| `allow-update { 10.10.0.30; }` | Autorise le serveur DHCP à mettre à jour le DNS dynamiquement |
| `allow-transfer { 10.10.0.30; }` | Autorise le transfert de zone vers cette IP |
| `notify yes` | Notifie les serveurs secondaires en cas de modification |

</details>

### Question 3 — Enregistrements pour déclarer le serveur DNS primaire dans la zone directe (4 pts)

<details>
<summary>Voir la réponse</summary>

```bash
$TTL 86400
@   IN  SOA  dns1.efm.local.  root.efm.local. (
            2023012001  ; Serial
            3600        ; Refresh
            1800        ; Retry
            604800      ; Expire
            86400 )     ; Minimum TTL

; Enregistrement NS — déclare le serveur de noms
@       IN  NS      dns1.efm.local.

; Enregistrement A — adresse IP du serveur DNS primaire
dns1    IN  A       10.10.0.1
```

Les enregistrements nécessaires sont :
- **NS** (Name Server) — indique quel serveur fait autorité pour la zone
- **A** (Address) — associe le nom `dns1` à l'adresse IP `10.10.0.1`

</details>

### Question 4 — Enregistrement pour déclarer le serveur de messagerie MAIL (IP `10.10.0.20`, priorité `30`) (2 pts)

<details>
<summary>Voir la réponse</summary>

```bash
; Enregistrement A — adresse du serveur mail
mail    IN  A       10.10.0.20

; Enregistrement MX — déclare le serveur de messagerie avec priorité 30
@       IN  MX  30  mail.efm.local.
```

:::info Priorité MX
La priorité `30` signifie que si plusieurs serveurs MX existent, celui avec la valeur la plus **basse** est préféré en premier. Une valeur de `10` serait prioritaire sur `30`.
:::

</details>

---

### Question 5 — Enregistrement pour l'alias `mail` au serveur de messagerie (2 pts)

<details>
<summary>Voir la réponse</summary>

```bash
; Enregistrement CNAME — alias "mail" pointant vers le serveur de messagerie
mail    IN  CNAME   mail.efm.local.
```

:::warning CNAME et MX
Un enregistrement CNAME ne peut pas pointer directement vers un enregistrement MX. L'alias `mail` pointera vers l'enregistrement A de `mail.efm.local` (`10.10.0.20`).
:::


</details>
