### Question 2 — Contenu du répertoire `/dev` sous Linux (1 pt)

<details>
<summary>Voir la réponse</summary>

Le répertoire `/dev` contient les **fichiers de périphériques** (device files) — des fichiers spéciaux qui représentent les composants matériels du système :

- **Disques durs** : `/dev/sda`, `/dev/sdb`...
- **Partitions** : `/dev/sda1`, `/dev/sda2`...
- **Terminaux** : `/dev/tty`, `/dev/pts`
- **Périphériques null/zero** : `/dev/null`, `/dev/zero`
- **Volumes RAID** : `/dev/md0`

Ces fichiers permettent aux programmes d'interagir avec le matériel comme avec des fichiers ordinaires.

</details>

---

### Question 3 — Expliquer la commande `mount /dev/sdc3 /media/stock` (1,5 pt)

<details>
<summary>Voir la réponse</summary>

Cette commande **monte** la partition `/dev/sdc3` sur le point de montage `/media/stock` :

- `/dev/sdc3` — 3ème partition du 3ème disque SCSI/SATA
- `/media/stock` — répertoire qui servira de point d'accès à la partition

Après cette commande, le contenu de la partition est accessible via `/media/stock`. Le montage est **temporaire** — il sera perdu au redémarrage si la ligne n'est pas ajoutée dans `/etc/fstab`.

</details>

---

### Question 4 — Définir le système RAID (1 pt)

<details>
<summary>Voir la réponse</summary>

**RAID** (Redundant Array of Independent Disks) est un ensemble de techniques de virtualisation du stockage permettant de répartir des données sur plusieurs disques pour améliorer soit les **performances**, soit la **tolérance aux pannes**, soit les deux.

Les niveaux principaux : RAID 0 (performances), RAID 1 (miroir), RAID 5 (parité répartie), RAID 6 (double parité), RAID 10 (miroir + stripe).

</details>


### Question 6 — Avantages des ACLs pour gérer les droits sous Linux (1 pt)

<details>
<summary>Voir la réponse</summary>

Les ACLs (Access Control Lists) offrent une **granularité supérieure** aux permissions Unix classiques (rwx) :

- Définir des droits pour **plusieurs utilisateurs ou groupes** sur le même fichier
- Appliquer des permissions différentes **sans modifier le groupe propriétaire**
- Gérer des cas complexes impossibles avec les 3 catégories classiques (propriétaire, groupe, autres)

```bash
setfacl -m u:user1:rw fichier.txt    # Donner rw à user1
getfacl fichier.txt                  # Afficher les ACLs
```

</details>

### Question 7 — Types de sauvegardes (1,5 pt)

<details>
<summary>Voir la réponse</summary>

**Complète (0,5 pt) :**
Sauvegarde de **l'intégralité des données** à un instant T, qu'elles aient été modifiées ou non. Restauration simple mais longue à réaliser et volumineuse.

**Incrémentale (0,5 pt) :**
Sauvegarde uniquement les données **modifiées depuis la dernière sauvegarde** (complète ou incrémentale). Rapide et légère, mais restauration complexe (nécessite la complète + toutes les incrémentales).

**Différentielle (0,5 pt) :**
Sauvegarde les données **modifiées depuis la dernière sauvegarde complète**. Plus volumineuse qu'une incrémentale mais restauration plus simple (complète + dernière différentielle uniquement).

</details>

### Question 8 — Rôle du service BIND (1 pt)

<details>
<summary>Voir la réponse</summary>

**BIND** (Berkeley Internet Name Domain) est le serveur DNS le plus utilisé sous Linux. Son rôle est de :

- Résoudre les **noms de domaine en adresses IP** (résolution directe)
- Résoudre les **adresses IP en noms de domaine** (résolution inverse)
- Gérer les **zones DNS** (zones directes et inversées)
- Transférer les zones vers les serveurs **DNS secondaires**

</details>

---

### Question 9 — Rôle du service HTTP (1 pt)

<details>
<summary>Voir la réponse</summary>

Le service **HTTP** (HyperText Transfer Protocol) permet d'héberger et de servir des **pages web** aux clients. Le serveur Apache (httpd) :

- Reçoit les requêtes HTTP/HTTPS des navigateurs clients
- Sert les fichiers HTML, CSS, JS, images depuis le `DocumentRoot`
- Gère les **VirtualHosts** pour héberger plusieurs sites sur la même machine
- Écoute par défaut sur le **port 80** (HTTP) et **443** (HTTPS)

</details>

**17. Compléter le fichier `/etc/named.conf`. (8 pts)**

Options : répertoire `/var/named` · écoute IPv4 et IPv6 · zone directe `NORTHIT.LAN` · zone inversée `172.16.1.0/24` · transfert vers SRVBIND2 · notifications vers SRVBIND2 · mises à jour dynamiques par SRVDHCP1 · récursion désactivée.

<details>
<summary>Voir la réponse</summary>

```bash
options {
    directory "/var/named";                          # a. Répertoire des zones
    listen-on port 53 { 172.16.1.50; };              # b. Écoute IPv4
    listen-on-v6 port 53 { any; };                   # b. Écoute IPv6
    allow-recursion { none; };                       # e. Désactiver récursion
};

# Zone directe
zone "NORTHIT.LAN" IN {
    type master;                                     # Zone principale
    file "NORTHIT.LAN.DIR";                          # Fichier de zone directe
    allow-transfer { 172.16.1.54; };                 # Transfert vers SRVBIND2
    notify yes;                                      # Notifications vers SRVBIND2
    allow-update { 172.16.1.53; };                   # MAJ dynamique par SRVDHCP1
};

# Zone inversée
zone "1.16.172.in-addr.arpa" IN {
    type master;                                     # Zone principale
    file "NORTHIT.LAN.INV";                          # Fichier de zone inversée
    allow-transfer { 172.16.1.54; };                 # Transfert vers SRVBIND2
    notify yes;                                      # Notifications vers SRVBIND2
    allow-update { 172.16.1.53; };                   # MAJ dynamique par SRVDHCP1
};
```

</details>

**19. Configurer le service DHCP sur `SRVDHCP1`. (6 pts)**

**a. Étendue DHCP :** réseau `172.16.1.0/24` · passerelle `172.16.1.1` · DNS `172.16.1.50` · domaine `northit.lan` · bail par défaut 3 jours · bail max 5 jours

**b. Exclure la plage `172.16.1.50` à `172.16.1.59`**

**c. Réservation pour `SRVSAMBA1` : MAC `02:04:06:A6:D9:C4` → IP `172.16.1.53`**

<details>
<summary>Voir la réponse</summary>

```bash
# /etc/dhcp/dhcpd.conf

ddns-update-style none;
default-lease-time 259200;      # 3 jours = 3 x 86400
max-lease-time 432000;          # 5 jours = 5 x 86400
authoritative;
log-facility local7;

# b. Exclure la plage 172.16.1.50 à 172.16.1.59
# (on définit des plages autour de la zone exclue)

subnet 172.16.1.0 netmask 255.255.255.0 {
    range 172.16.1.1 172.16.1.49;
    range 172.16.1.60 172.16.1.254;
    option subnet-mask 255.255.255.0;
    option domain-name-servers 172.16.1.50;
    option routers 172.16.1.1;
    option domain-name "northit.lan";
}

# c. Réservation pour SRVSAMBA1
host SRVSAMBA1 {
    hardware ethernet 02:04:06:A6:D9:C4;
    fixed-address 172.16.1.53;
}
```

Démarrer le service :

```bash
sudo systemctl restart dhcpd
sudo systemctl enable dhcpd
systemctl status dhcpd
```

</details>

**21. Compléter la configuration du VirtualHost Apache. (2 pts)**

Paramètres : IP `172.16.1.52` · port `81` · nom `www.northit.lan` · racine `/var/html/northit.ma`

<details>
<summary>Voir la réponse</summary>

```apache
<VirtualHost 172.16.1.52:81>
    ServerName www.northit.lan
    ServerAlias northit.lan
    DocumentRoot /var/html/northit.ma
</VirtualHost>
```

:::info Port non standard
Le port `81` est non standard — il faut aussi ajouter `Listen 81` dans `/etc/httpd/conf/httpd.conf` ou `/etc/apache2/ports.conf` pour qu'Apache écoute sur ce port.
:::

Après modification :

```bash
sudo apachectl configtest
sudo systemctl restart httpd
```

</details>
