## Partie Théorique (16 pts)

### Q1 — Avantage principal de LVM par rapport aux partitions traditionnelles : (2 pts)

- A. LVM permet la création de partitions plus petites
- B. LVM permet la création de partitions plus grandes
- C. **LVM offre la flexibilité de redimensionner les volumes logiques à chaud**
- D. LVM garantit une meilleure performance des disques durs

<details>
<summary>Voir la réponse</summary>

**Réponse : C — LVM offre la flexibilité de redimensionner les volumes logiques à chaud**

Contrairement aux partitions classiques, les volumes LVM peuvent être étendus ou réduits sans démonter le système ni redémarrer le serveur. C'est l'avantage majeur de LVM en production.

</details>

### Q2 — Commande Linux pour créer un nouveau volume logique avec LVM : (2 pts)

- A. **lvcreate**
- B. lvextend
- C. lvresize
- D. lvremove

<details>
<summary>Voir la réponse</summary>

**Réponse : A — lvcreate**

```bash
sudo lvcreate -L 10G -n monvolume monvg
```

| Commande | Rôle |
|---|---|
| `lvcreate` | Créer un nouveau volume logique |
| `lvextend` | Étendre un volume logique existant |
| `lvresize` | Redimensionner (étendre ou réduire) |
| `lvremove` | Supprimer un volume logique |

</details>

### Q3 — Quelle affirmation décrit correctement le RAID 0 ? (2 pts)

- A. Le RAID 0 offre une redondance des données
- B. **Le RAID 0 divise les données en blocs et répartit sur plusieurs disques**
- C. Le RAID 0 utilise la technique du mirroring

<details>
<summary>Voir la réponse</summary>

**Réponse : B — Le RAID 0 divise les données en blocs et les répartit sur plusieurs disques (striping)**

Le RAID 0 améliore les performances en lecture/écriture mais n'offre **aucune redondance** — si un disque tombe, toutes les données sont perdues.

</details>

---

### Q4 — Caractéristique principale du RAID 1 en termes de redondance : (2 pts)

- A. Aucune redondance, les données sont distribuées de manière fragmentée
- B. **Redondance totale, chaque donnée est écrite sur au moins deux disques**
- C. Redondance partielle, certaines parties des données sont répliquées

<details>
<summary>Voir la réponse</summary>

**Réponse : B — Redondance totale, chaque donnée est écrite sur au moins deux disques**

Le RAID 1 (mirroring) copie les données identiquement sur deux disques ou plus. Si un disque tombe en panne, les données sont intégralement disponibles sur le miroir.

</details>

### Q5 — Rôle du service Samba : (2 pts)

<details>
<summary>Voir la réponse</summary>

**Samba** est un service de partage de fichiers et d'imprimantes qui implémente le protocole **SMB/CIFS** sous Linux. Il permet :

- Le **partage de répertoires** Linux accessibles depuis des clients Windows, macOS et Linux
- Le **partage d'imprimantes** réseau
- L'**intégration dans un domaine Active Directory** ou un groupe de travail Windows
- L'authentification des utilisateurs via des comptes Samba

```bash
# Fichier de configuration principal
/etc/samba/smb.conf

# Exemple de partage
[partage]
    path = /srv/partage
    browseable = yes
    read only = no
    valid users = user1 user2
```

</details>

### Q6 — Rôle de la commande `ssh-copy-id` : (2 pts)

<details>
<summary>Voir la réponse</summary>

`ssh-copy-id` copie automatiquement la **clé publique SSH** de l'utilisateur local vers le fichier `~/.ssh/authorized_keys` d'un serveur distant.

```bash
ssh-copy-id user@192.168.1.50
```

Après cette commande, l'utilisateur peut se connecter au serveur **sans mot de passe**, en utilisant sa clé privée pour l'authentification.

Prérequis : avoir généré une paire de clés avec `ssh-keygen`.

</details>

### Q7 — Directive `anon_upload_enable YES` dans vsftpd : (1 pt)

<details>
<summary>Voir la réponse</summary>

Cette directive **autorise les utilisateurs anonymes à envoyer (uploader) des fichiers** sur le serveur FTP.

:::danger Risque de sécurité
`anon_upload_enable YES` est très risqué en production — n'importe qui peut déposer des fichiers sur le serveur sans authentification. À n'utiliser que dans des environnements contrôlés et isolés.
:::

</details>

### Q8 — Directive `PermitRootLogin NO` dans `sshd_config` : (1 pt)

<details>
<summary>Voir la réponse</summary>

Cette directive **interdit la connexion SSH directe avec le compte root**.

C'est la configuration **recommandée en production** — les administrateurs doivent d'abord se connecter avec un compte utilisateur standard, puis utiliser `sudo` pour obtenir les privilèges root.

```bash
# Dans /etc/ssh/sshd_config
PermitRootLogin no

# Appliquer
sudo systemctl restart sshd
```

</details>

**Q5. Configurer le serveur DHCP selon les paramètres donnés. (4 pts)**

Paramètres : sous-réseau `192.168.1.0/24` · plage `192.168.1.20` à `192.168.1.200` · passerelle `192.168.1.1` · DNS `192.168.1.2` · domaine `exam.ma`

<details>
<summary>Voir la réponse</summary>

```bash
# /etc/dhcp/dhcpd.conf

default-lease-time 600;
max-lease-time 7200;
authoritative;

subnet 192.168.1.0 netmask 255.255.255.0 {
    range 192.168.1.20 192.168.1.200;
    option routers 192.168.1.1;
    option domain-name-servers 192.168.1.2;
    option domain-name "exam.ma";
}
```

Démarrer le service :

```bash
# Fedora
sudo systemctl restart dhcpd
sudo systemctl enable dhcpd

# Ubuntu
sudo systemctl restart isc-dhcp-server
sudo systemctl enable isc-dhcp-server
```

</details>

**Q6. Réservation de `192.168.1.20` pour `SRV-WEB` (MAC : `26-77-03-F1-E1-C1`). (3 pts)**

<details>
<summary>Voir la réponse</summary>

Dans `/etc/dhcp/dhcpd.conf`, ajouter après le bloc subnet :

```bash
host SRV-WEB {
    hardware ethernet 26:77:03:F1:E1:C1;
    fixed-address 192.168.1.20;
}
```

:::info
Les tirets `-` de l'adresse MAC dans le sujet se remplacent par des deux-points `:` dans la configuration DHCP Linux.
:::

</details>

**Q8. Créer un VirtualHost pour `www.exam.ma` — dossier `/var/www/site1` — port `80`. (3 pts)**

<details>
<summary>Voir la réponse</summary>

Créer le fichier `/etc/httpd/conf.d/exam.conf` (Fedora) ou `/etc/apache2/sites-available/exam.conf` (Ubuntu) :

```apache
<VirtualHost *:80>
    ServerName www.exam.ma
    DocumentRoot /var/www/site1
    ServerAdmin admin@exam.ma
    ErrorLog /var/log/httpd/exam-error.log
    CustomLog /var/log/httpd/exam-access.log combined
</VirtualHost>
```

Créer le répertoire et le fichier index :

```bash
sudo mkdir -p /var/www/site1
echo "<h1>www.exam.ma</h1>" | sudo tee /var/www/site1/index.html

# Vérifier la syntaxe
sudo apachectl configtest

# Ubuntu uniquement — activer le site
sudo a2ensite exam.conf
sudo systemctl reload apache2
```

</details>
